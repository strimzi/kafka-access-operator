SUMMARY OF CHANGES - Dual Namespace Environment Variables
===========================================================

## Overview
Implemented two separate environment variables to control namespace watching:
- ACCESS_WATCHED_NAMESPACES: for KafkaAccess resources and resulting Secrets
- KAFKA_WATCHED_NAMESPACES: for Kafka cluster resources (Kafka, KafkaUser, Secrets)

   - Added environment variable ACCESS_WATCHED_NAMESPACES (defaults to "*")

## Architecture Summary

Old Architecture:
- Single ClusterRole with all permissions
- Single ClusterRoleBinding granting cluster-wide access
- One STRIMZI_NAMESPACE environment variable

New Architecture:
- Two ClusterRoles (access-resources and kafka-resources)
- Two namespace-scoped RoleBindings (one per namespace)
- Two environment variables for granular namespace control
- Least-privilege principle: read-only for Kafka resources, CRUD only for KafkaAccess
- Operator can watch different namespaces for access vs kafka resources
